It only looks at what changed.

Most of the time it does nothing, and costs nothing. When a repo actually changes, it checks just the new code, and acts only where it's safe.

  1. Changed repos: a cheap API check spots which repos changed, spending zero tokens.
  2. Just the new diff: it reads only the new code, never the whole tree.
  3. Scan and classify: it scans the diff and sorts findings into tiers 1, 2, and 3.
  4. Tier 1, safe fix: patch and minor bumps that pass CI auto-merge.
  5. Tier 2, needs a look: code findings open a pull request for your review.
  6. Tier 3, secret: a leaked credential is alert-only and never auto-touched.

Two layers. One quiet helper.

GitHub's own engine handles the dependency bulk. A small, careful agent handles the rest.

Layer 1 · native

Dependabot, everywhere

Turns on GitHub's built-in vulnerability scanning and auto-fix pull requests across every one of your repos at once. Patch and minor bumps that pass CI merge themselves; bigger ones wait for you. Runs in GitHub's cloud, zero upkeep.

Enabled in one command
Layer 2 · the agent

The incremental watcher

On a schedule, it checks only the repos that changed, reads only the new code, scans it, and tiers what it finds. Safe fixes go out as PRs that auto-merge on green; anything ambiguous waits for your review; secrets are never touched, only flagged.

Ships dry-run safe · you enable repos one at a time

Careful by design.

Every default leans toward safe. The interesting decisions are always yours.

Costs nothing at rest

When no repo has changed, it does nothing and makes no PRs. The watching is a cheap API check; the scan only runs on real deltas.

Tiered, never reckless

Only patch and minor dependency bumps that pass CI ever merge on their own. Code findings become a PR you review. Nothing surprising ships unattended.

Secrets stay yours

A leaked credential is never auto-fixed. The bot flags it and stops, because rotation is a decision only you should make. No exceptions.

Its own scoped identity

Picket acts as a least-privilege GitHub App, not your personal token. You grant it exactly what it needs, see every action, and can revoke it in one click.
